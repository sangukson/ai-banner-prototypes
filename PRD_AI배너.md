# PRD: AI 배너 (콘텐츠 스튜디오)

**상태**: 기획 진행 중 (프/테 완료 → 개발 설계 단계)  
**베팅**: Big Batch — 1주(프/테) + 3주(개발)  
**최종 수정**: 2026-04-16

---

## 한 줄 요약
배너 주제 선정 → 제품 이미지 업로드 → 레퍼런스 선택 → AI 생성 → 스킨 적용까지 원스톱 워크플로우.  
사장님이 디자인을 "정의"하는 게 아니라 **"선택"만 하면 됨**.

---

## 배너 유형 (미리 알고가기)

| 유형 | 설명 |
|------|------|
| 히어로 배너 | 메인페이지 최상단, 쇼핑몰 첫인상 결정 |
| 2단 콘텐츠 배너 | 범용혜택 안내 (무료배송, 환불보장 등) |
| 1단 콘텐츠 배너 | 혜택 안내 / 특정 신상품 안내 |

## 1. 문제

### 핵심: 템플릿의 역설
고객은 완벽하게 세팅된 스킨을 보고 구매하지만, 실제 운영 단계에서 **비주얼 에셋 격차**를 극복하지 못해 기대한 퀄리티를 구현하지 못한다.

### 현재 시장의 해결 방식과 한계

| 플레이어 | 방식 | 문제점 |
|----------|------|--------|
| 대부분 에이전시 | 배너 원본 PSD 파일 제공 | 포토샵 학습 필요 + 좋은 제품사진 없으면 결과 낮음 |
| 디자인플로어 | 배너 데스크 기능 | 배경 이미지 따로 구해야 함 + HTML 학습 필요 |

### 더 나아가: 제작 과정 자체도 고통
- "뭐가 예쁜 건지 모르겠다" — 좋은 디자인 기준 자체 없음
- "우리 상품에 뭐가 어울리는지 모르겠다" — 브랜드 맞는 조합 판단 불가
- "이게 맞나?" — 감각 아닌 찍기로 만들고 → 지우고 → 반복 루프

### 왜 중요한가
- **The 0.05s Rule**: 사용자가 첫인상 결정에 0.05초 소요
- 메인페이지 디자인 퀄리티 붕괴 → 상품까지 의심 → 브랜드 신뢰도 하락

## 2. 솔루션

**콘텐츠 스튜디오**: 배너 제작 전체 워크플로우를 원스톱으로 진행.

### As-is → To-be

| | 상세 | 소요시간 | 퀄리티 |
|---|---|---|---|
| **As-is** | 어떤 배너 필요한지 파악 → 레퍼런스 직접 검색 → 포토샵/캔바 학습 → 제품사진 촬영·보정 → 디자인 제작(시행착오) → 배너매니저 수동 업로드 | 2시간+ | 역량에 따라 천차만별 |
| **To-be** | 추천 배너 주제 확인 → 제품사진 업로드 → 레퍼런스 선택 → AI 제작 → 원클릭 스킨 적용 | 5~10분 | 일관되게 높은 수준 |

## 3. 유저 시나리오 (6단계)

**Step 1. 업체 기본 정보 확인**  
온보딩 수집 정보 자동 불러오기 (없으면 간단 질문)

**Step 2. 배너 목적 선택**  
`제품/카테고리 홍보` / `기획전/프로모션 홍보` / `일반 혜택 홍보`
- 일반 혜택: 사전 제작 이미지 다운로드로 분기 (예: 신규 회원가입 쿠폰, 무료배송)

**Step 3. 제품 이미지 업로드**  
고퀄리티 불필요. 스마트폰 촬영 수준이면 충분.  
권고: 배경이 복잡하지 않은 선명한 이미지, 최소 800×800

**Step 4. 레퍼런스 탐색 & 스타일 선택**  
업종·카테고리 사전 필터링된 레퍼런스 제시 → 선택만 하면 됨

**Step 5. AI 배너 생성**  
레퍼런스 스타일 + 제품 이미지 + 텍스트 카피 → AI 생성  
- 최초 생성: 3장 고정  
- 이후 재생성: 1~3장 선택 가능 (선택한 장수만큼 크레딧 차감)  
PC 기준 생성 → MO 리사이징

**Step 6. 결과 확인 & 적용**

| 상황 | 액션 |
|------|------|
| 마음에 든다 + 알파캔버스 스킨 사용 | 스킨 해당 슬롯에 자동 삽입 |
| 마음에 든다 + 스킨 미사용 | PNG/JPG 다운로드 |
| 마음에 안 든다 | 레퍼런스 선택부터 다시 (새 생성) |

> **제거된 재생성 옵션 (2026-04-16 결정)**  
> - ~~같은 레퍼런스로 다시 뽑기~~: 랜덤 씨드 재시도 개념으로 초보 사장님에게 불명확  
> - ~~특정 이미지 1개 추가 프롬프팅~~: 이미지별 재생성과 동일 컨셉으로 중복  
> → 재생성은 "레퍼런스 선택부터 다시" 단일 경로로 단순화

**Step 7. 생성 이미지 보관함**  
생성된 배너는 **3일간 자동 보관**. 유저는 언제든지 배너 보관함에서 확인·재다운로드 가능.  
3일 경과 시 자동 삭제.

## 5. 크레딧 정책

**기본 규칙**: 이미지 1장 생성 = 1 Credit 차감  
**비용**: 1장 $0.16 / 1회(3장) $0.48

**생성 개수 규칙** (2026-04-16 확정)
- 최초 생성: 3장 고정 (선택 불가)
- 재생성: 1~3장 유저가 선택 → 선택한 장수만큼 크레딧 차감
- 최솟값 1장 / 최댓값 3장

| 구분 | 무료 제공량 | 최초 생성 후 잔여 | 비고 |
|------|------------|-----------------|------|
| 기본 제공 (전체) | **10 Credits** | 7 Credits (재생성 최대 7회×1장) | 스킨 구매 여부 무관 |
| 스킨 구매 업체 | 50 Credits | 47 Credits | (별도 논의 중) |

> 기본 제공 10 Credits는 스킨 구매 여부와 관계없이 제공. 유료 전환 정책은 4월 막주 이후 코어팀 논의.

**경쟁사 비교**
- 미리캔버스: 10회 (평생)
- 젠스파크: 10~20장/일
- Gemini: ~100장/일
- ChatGPT DALL-E 3: 2~3장/일

## 9. FAQ (리스크 대응)

**Q. 레퍼런스 이미지는 어떻게 구하나?**  
업종별 최소 10~15개 필요 → 총 80~150개 예상.  
2026-04-06 주간 AI 생성 시도 → 안 되면 외주 전환.

**Q. 무료 토큰 사용량이 너무 많으면?**  
500개 업체 전원 무료 소진 시 약 120만원. 개수 제한으로 비용 통제.

**Q. 이미지 퀄리티 보장?**  
레퍼런스 기반 생성(하한선 확보) + 프리뷰·재생성(사전 필터링) + 내부 모니터링

**Q. 저퀄리티 제품 사진이면?**  
최소 800×800 가이드 + 경고 메시지. 스마트폰 기본 카메라 수준이면 처리 가능하도록.

**Q. 미리캔버스/캔바와 차별성?**  
쇼핑몰 특화 레퍼런스 + 올인원 워크플로우 (생성 → 스킨 세팅까지)

## 10. 성공 지표

| 지표 | 정의 | 측정 시점 | 목표 |
|------|------|-----------|------|
| 배너 실적용률 | 생성 업체 중 실제 스킨 세팅/다운로드한 비율 | 배포 후 4주 (20개사 미달 시 연장) | 70% 이상 |
| 배너 스튜디오 시도율 | 스킨 구매 업체 중 1회 이상 시도한 비율 | 배포 후 4주 | 50% 이상 |

**기타 트래킹**
- 배너 생성 → 스킨 적용까지 평균 10분 이내
- 재생성 횟수 업체당 평균 2회 이하 (배너 실적용 업체 기준)
- 정성: "외주 안 맡기고 직접 할 수 있겠다" 반응
